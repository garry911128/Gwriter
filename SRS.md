## 1. 簡介

### 1.1 目的

本系統之研發旨在建立一個對作家友善的小說編輯器及更多元化的小說閱讀器，主要目標包含：

1. **核心創作編輯工具：** 提供作家專業且直覺的小說編輯器，除支援基礎的章節管理與草稿自動保存外，更進一步整合人物設定、場景佈局等多元創作設定。這能協助作家在創作過程中快速鎖定敘事重點，並透過友善的小說管理介面，輕鬆掌握作品的整體結構與發布狀態。

2. **沉浸式多元閱讀平台：** 突破傳統純文字閱讀模式，讓讀者能根據作家提供的選項自由切換閱讀體驗。系統支援高度自定義的閱讀環境設定，包含背景音樂 (BGM)、環境場景模擬、特定句子的動態音效，以及即時查看人物卡片等功能。透過這些多媒體元素的整合，提升讀者在閱讀過程中的感官互動與代入感。

3. **商業擴展框架：** 建構完整的帳務與交易基礎設施，支援未來的小說購買、營業收入追蹤及買賣紀錄管理，實現作品的商業價值。

4. **品質反饋機制：** 提供基礎的評論與評分功能，作為讀者與作家間的溝通橋樑，協助作家根據反饋優化內容。

### 1.2 範圍

本專案將開發一個創新小說平台，旨在平衡作家的深度創作需求與讀者的感官閱讀體驗。

#### In-Scope
- 使用者註冊、登入與基本帳號管理
- 小說瀏覽、搜尋與分類查詢
- 小說購買、觀看紀錄與書架管理
- 作者端小說上傳與管理（含人物及場景設定等創作工具）
- 使用者對小說進行評論與評分
- 平台營業收入與銷售紀錄查詢
- 第三方支付

#### Out-of-Scope
- 實體書籍的印刷與物流配送
- 即時通訊或社交聊天功能
- 第三方支付以外之金融服務
- 跨平台行動裝置原生應用程式（僅限 Web 版本）

### 1.3 定義與縮寫

| 術語 | 定義 |
|------|------|
| Character Card | 作者在編輯器中所建立的結構化人物設定資料庫。包含人物姓名、圖片（頭像或立繪）、文字簡介及人物間的關係圖等資訊，並可供讀者在閱讀過程中即時查看 |
| Scene Card | 作者在創作時針對特定章節或段落所設定的視覺/文字描述等呈現方式，包含背景環境模擬與視覺特效之配置 |
| BGM | Background Music，背景音樂。指作者為小說章節設定的背景配樂，旨在營造特定的故事氛圍 |
| WYSIWYG | What You See Is What You Get，即時預覽的富文字編輯器 |
| JWT | JSON Web Token，用於使用者身份驗證的加密令牌 |
| API | Application Programming Interface，應用程式介面 |
| PWA | Progressive Web App，漸進式網頁應用程式 |

## 2. 系統

### 2.1 系統描述與系統架構

#### 2.1.1 系統概述

GWriter 採用三層式架構（Three-Tier Architecture），分為前端展示層、後端應用層與資料庫層，並透過 Docker 容器化部署。

```
┌─────────────────────────────────────────────────────────────┐
│                        使用者端                              │
│                  瀏覽器 / PWA 桌面捷徑                       │
└─────────────────────────┬───────────────────────────────────┘
                          │ HTTPS
┌─────────────────────────▼───────────────────────────────────┐
│                     前端展示層                               │
│                                                             │
│   React 18 + TypeScript + Vite                             │
│   ├── 小說編輯器 (NovelEditor)                              │
│   │   ├── 章節管理 (ChaptersList)                           │
│   │   ├── WYSIWYG 編輯器 (React Quill)                     │
│   │   ├── 人物設定 (CharactersList)                         │
│   │   ├── 世界設定 (WorldList)                              │
│   │   └── AI 助手 (AIAssistant)                            │
│   ├── 讀者閱讀平台                                          │
│   ├── 書架管理                                              │
│   └── 商店 / 支付頁面                                       │
│                                                             │
│   Tailwind CSS + Bootstrap 5 + Lucide Icons                │
│   部署：Nginx (Docker)                                      │
└─────────────────────────┬───────────────────────────────────┘
                          │ HTTP REST API (port 8080)
┌─────────────────────────▼───────────────────────────────────┐
│                     後端應用層                               │
│                                                             │
│   Go 1.21 + Gorilla Mux                                    │
│   ├── /api/v1/novels        — 小說 CRUD                    │
│   ├── /api/v1/chapters      — 章節 CRUD + Auto-save        │
│   ├── /api/v1/characters    — 人物設定                     │
│   ├── /api/v1/world         — 世界設定                     │
│   ├── /api/v1/users         — 用戶管理 + JWT 認證          │
│   ├── /api/v1/comments      — 評論管理                     │
│   └── /api/v1/orders        — 訂單 / 金流                  │
│                                                             │
│   中介層：CORS Middleware, JWT Auth Middleware              │
│   部署：Go binary (Docker)                                  │
└─────────────────────────┬───────────────────────────────────┘
                          │ MySQL Driver (port 3306)
┌─────────────────────────▼───────────────────────────────────┐
│                      資料庫層                                │
│                                                             │
│   MySQL 5.7                                                 │
│   ├── users          — 用戶帳號                             │
│   ├── novels         — 小說基本資訊                         │
│   ├── chapters       — 章節內容                             │
│   ├── characters     — 人物設定                             │
│   ├── world_items    — 世界設定                             │
│   ├── drafts         — 草稿版本                             │
│   ├── comments       — 評論                                 │
│   └── orders         — 訂單紀錄                             │
│                                                             │
│   部署：MySQL Docker Container，Volume 持久化              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 操作概念 (User Story)

#### 2.2.1 Scenario 1: Guest Operational Concepts

| 需求 | 描述 |
|------|------|
| GUS-01 | 作為一位訪客，我希望能夠瀏覽小說網站的首頁，看到熱門和最新上架的小說，以便快速找到感興趣的小說。 |
| GUS-02 | 作為一位訪客，我希望能夠查看每本小說的詳細資訊，包括簡介、章節列表和評論數量，以便了解是否適合閱讀。 |
| GUS-03 | 作為一位訪客，我希望能夠通過搜索框和分類篩選小說，以便更快找到感興趣的小說。 |
| GUS-04 | 作為一位訪客，我希望能夠註冊成為會員，以便獲得更多的網站功能，如收藏、評論和小說創作。 |

#### 2.2.2 Scenario 2: Member Operational Concepts

| 需求 | 描述 |
|------|------|
| MUS-01 | 作為一位讀者會員，我希望能夠收藏喜歡的小說，以便稍後可以快速找到並繼續閱讀。 |
| MUS-02 | 作為一位讀者會員，我希望能夠對小說進行評論，以便分享我的閱讀心得，並與其他讀者互動。 |
| MUS-03 | 作為一位讀者會員，我希望能夠對小說進行評分，以便幫助其他讀者了解小說的受歡迎程度。 |
| MUS-04 | 作為一位讀者會員，我希望能夠查看自己的閱讀歷史，以便隨時回到上次閱讀的地方。 |

| MUS-06 | 作為一位作家會員，我希望能夠查看讀者對我作品的評論，以便了解讀者的反饋，並改善我的寫作內容。 |

| MUS-08 | 作為一位作家會員，我希望能夠查看我的作品的瀏覽量和收藏數量，以便了解作品的受歡迎程度。 |

### 2.3 功能性需求 (Functional Requirements)

#### 1. 小說管理 (Novel Management)

| 編號 | 功能 | 描述 |
|------|------|------|
| NM-01 | 創建小說 | 系統應允許作家創建新的小說，並提供標題、簡介和封面圖片等基本資訊填寫。作家可以為小說添加章節並編輯內容。 |
| NM-02 | 編輯小說 | 系統應允許作家修改小說的標題、簡介、封面圖片和已存在的章節。作家可以在小說草稿中進行無限制的編輯，直至準備發布。 |
| NM-03 | 刪除小說 | 系統應允許作家刪除自己的小說，刪除後小說將從網站上消失，無法再被讀者訪問。 |
| NM-04 | 查看小說詳情 | 讀者和作家可查看小說的詳細頁面，包括小說簡介、章節列表、評論區等。 |
| NM-05 | 查看小說章節 | 系統應允許讀者在小說詳細頁面中查看小說的所有章節，並根據需要進行翻閱。 |
| NM-06 | 發布小說 | 作家可以將編寫完成的小說公開發布，使其對所有用戶可見。 |
| NM-07 | 編輯小說標籤 | 作家/讀者可以新增/刪除小說的標籤。 |

#### 2. 章節管理 (Chapter Management)

| 編號 | 功能 | 描述 |
|------|------|------|
| CM-01 | 創建章節 | 作家可以為小說創建新章節，並輸入章節標題和內容。 |
| CM-02 | 編輯章節 | 作家可以修改已創建的章節內容，包括標題和正文內容。支援 WYSIWYG 富文字編輯器。 |
| CM-03 | 刪除章節 | 作家可以刪除某個章節，刪除後該章節將不再顯示在小說中。 |
| CM-04 | 章節排序 | 系統應允許作家對章節進行排序，以確保章節按照正確的順序顯示。 |
| CM-05 | 自動儲存 | 編輯器應在作家停止輸入後 3 秒自動儲存章節內容至伺服器，並顯示儲存狀態。 |
| CM-06 | 字數統計 | 系統應即時顯示當前章節及整部小說的總字數，並提供每日目標字數進度。 |

#### 3. 人物與世界設定 (Character & World Management)

| 編號 | 功能 | 描述 |
|------|------|------|
| CW-01 | 建立人物卡片 | 作家可建立人物設定，包含姓名、角色定位、外觀描述及頭像圖片。 |
| CW-02 | 編輯/刪除人物 | 作家可修改或移除已建立的人物設定。 |
| CW-03 | 讀者查看人物卡片 | 讀者在閱讀過程中可即時查看人物資訊卡片。 |
| CW-04 | 建立世界設定 | 作家可建立世界觀設定，分類為地點 (location)、歷史 (history)、文化 (culture)。 |
| CW-05 | 編輯/刪除世界設定 | 作家可修改或移除已建立的世界設定條目。 |

#### 4. 評論管理 (Comment Management)

| 編號 | 功能 | 描述 |
|------|------|------|
| COM-01 | 發表評論 | 讀者可以在小說詳細頁面上發表評論，並對評論進行回覆。 |
| COM-02 | 刪除評論 | 讀者可以刪除自己發表的評論，管理員可以刪除不適當的評論。 |
| COM-03 | 評論點讚 | 讀者可以對其他人的評論進行點讚，這會增加該評論的可見度。 |
| COM-04 | 查看評論 | 讀者可以查看每本小說的評論區，並查看其他讀者的評論。 |

#### 5. 用戶管理 (User Management)

| 編號 | 功能 | 描述 |
|------|------|------|
| UM-01 | 註冊帳號 | 新用戶可以註冊帳號，並提供必須的基本資料如電子郵件和密碼。 |
| UM-02 | 登入帳號 | 用戶可以使用註冊的帳號登錄系統，系統使用 JWT Token 維持登入狀態。 |
| UM-03 | 修改帳號資料 | 用戶可以修改自己的個人資料，如用戶名、密碼、電子郵件等。 |
| UM-04 | 角色管理 | 系統應區分訪客、讀者、作家、管理員的角色，根據角色授予不同的權限。 |

#### 6. 搜索與推薦 (Search & Recommendation)

| 編號 | 功能 | 描述 |
|------|------|------|
| SR-01 | 小說搜尋 | 讀者可以使用關鍵字、標籤、作者名稱等條件來搜索小說。 |
| SR-02 | 熱門推薦 | 系統根據閱讀數與收藏數推薦熱門小說。 |
| SR-03 | 最新上架 | 系統顯示最新上架的小說，供讀者瀏覽和閱讀。 |
| SR-04 | 分類推薦 | 系統可以根據小說的分類（如科幻、奇幻、懸疑等）或標籤推薦相關作品。 |

#### 7. 書架管理 (Bookshelf Management)

| 編號 | 功能 | 描述 |
|------|------|------|
| BM-01 | 加入書架 | 讀者可以將喜歡的小說加入自己的書架，以便快速訪問。 |
| BM-02 | 管理書架 | 讀者可以查看書架中所有已收藏的小說，並根據需要添加、刪除或重新排序。 |
| BM-03 | 閱讀紀錄 | 系統應記錄讀者的閱讀歷史，並允許讀者查看過去閱讀過的小說和章節。 |

#### 8. 草稿管理 (Draft Management)

| 編號 | 功能 | 描述 |
|------|------|------|
| DM-01 | 保存草稿 | 作家可以隨時保存正在編寫的小說草稿，並稍後繼續編輯。 |
| DM-02 | 草稿列表 | 作家可以查看自己所有的草稿，並選擇繼續編輯或發布。 |
| DM-03 | 版本回溯 | 系統保存章節的歷史版本，作家可回溯至先前的草稿版本。 |

#### 9. 沉浸式閱讀 (Immersive Reading)

| 編號 | 功能 | 描述 |
|------|------|------|
| IR-01 | BGM 播放 | 系統支援作者為章節設定背景音樂，讀者閱讀時自動播放。 |
| IR-02 | 場景模擬 | 系統支援作者設定章節的視覺背景場景（如天空、森林、城市等）。 |
| IR-03 | 動態音效 | 特定段落可觸發環境音效（如雨聲、戰鬥音效）。 |
| IR-04 | 閱讀設定 | 讀者可自定義字體大小、行距、背景色（日間/夜間模式）。 |

#### 10. 收入與買賣紀錄 (Sales & Income)

| 編號 | 功能 | 描述 |
|------|------|------|
| SI-01 | 查看營業收入 | 管理員和作家可查看每本小說的銷售收入，並分析營收數據。 |
| SI-02 | 查看買賣紀錄 | 用戶可以查看自己購買的小說記錄，包括日期和金額。 |
| SI-03 | 第三方支付 | 系統整合第三方支付（如綠界、街口支付），處理小說購買。 |

### 2.4 非功能性需求 (Non-Functional Requirements)

#### 2.4.1 性能需求 (Performance)

| 編號 | 需求 | 指標 |
|------|------|------|
| PF-01 | 同時在線人數 | 系統應在高峰時段能夠處理至少 5,000 名同時在線使用者 |
| PF-02 | 頁面載入時間 | 前端頁面在正常網速（100Mbps）的情況下應在 2 秒內加載完成 |
| PF-03 | API 回應時間 | 後端 API 的回應時間不應超過 500 毫秒 |
| PF-04 | 資料庫查詢 | 資料庫查詢操作（特別是小說搜尋與篩選）應在 1 秒內返回結果 |
| PF-05 | 可擴展性 | 系統設計應能夠水平擴展，以適應未來的流量增長 |

#### 2.4.2 安全需求 (Security)

| 編號 | 需求 | 實作方式 |
|------|------|------|
| SEC-01 | 身份驗證 | 所有用戶必須通過 JWT Token 進行身份驗證 |
| SEC-02 | 密碼加密 | 用戶密碼使用 Bcrypt 強加密算法儲存 |
| SEC-03 | 傳輸加密 | 前後端通信使用 HTTPS 確保資料傳輸的安全性 |
| SEC-04 | 防 SQL Injection | 後端 API 使用參數化查詢，防止 SQL 注入攻擊 |
| SEC-05 | CORS 控制 | API 設定適當的 CORS 政策，限制非授權來源的請求 |
| SEC-06 | 角色授權 | 根據用戶角色（訪客/讀者/作家/管理員）限制可存取的 API 端點 |

#### 2.4.3 可用性需求 (Usability)

| 編號 | 需求 | 指標 |
|------|------|------|
| UB-01 | 導航效率 | 使用者能夠在三次點擊內找到想要的小說或資訊 |
| UB-02 | 錯誤處理 | 系統應提供清晰的錯誤信息，並對常見錯誤提供友好的提示或指引 |
| UB-03 | 跨瀏覽器支援 | 系統前端應兼容 Chrome、Firefox、Safari 主流瀏覽器 |
| UB-04 | 響應式設計 | 介面應適應桌面和移動設備的不同螢幕尺寸 |
| UB-05 | 深色模式 | 編輯器與閱讀器均支援日間/夜間模式切換 |

#### 2.4.4 可維護性需求 (Maintainability)

| 編號 | 需求 | 說明 |
|------|------|------|
| MT-01 | 模組化設計 | 前端和後端應以模組化方式設計，以便單獨修改或擴展功能 |
| MT-02 | 程式碼規範 | 前端遵循 ESLint 規範，後端遵循 Go 編碼風格指引 |
| MT-03 | 日誌管理 | 後端應實現錯誤日誌、請求日誌和事件日誌 |
| MT-04 | API 版本控制 | API 路徑使用 `/api/v1/` 前綴，便於未來版本升級 |

#### 2.4.5 可移植性需求 (Portability)

| 編號 | 需求 | 說明 |
|------|------|------|
| PT-01 | 容器化 | 系統打包成 Docker 容器，確保開發/測試/正式環境一致性 |
| PT-02 | 環境配置 | 透過環境變數（.env）支援多種環境配置 |
| PT-03 | 跨平台後端 | Go 後端能在 Linux 系統穩定運行，支援主流雲端平台部署 |

#### 2.4.6 可回復性需求 (Recoverability)

| 編號 | 需求 | 說明 |
|------|------|------|
| RC-01 | 資料備份 | 資料庫應設定每日自動備份，並保存最近 30 天的數據 |
| RC-02 | 錯誤回滾 | 系統發生錯誤或更新失敗時，應能迅速回滾至上一穩定版本 |
| RC-03 | 崩潰恢復 | 系統應能在崩潰後自動重新啟動服務（Docker restart policy） |

#### 2.4.7 可測試性需求 (Testability)

| 編號 | 需求 | 指標 |
|------|------|------|

| TS-02 | 集成測試 | 應定期進行前後端整合測試，確保 API 和前端界面的協同工作 |
| TS-03 | 自動化測試 | 系統應實現基本的自動化測試腳本，於每次部署時進行功能回歸測試 |
